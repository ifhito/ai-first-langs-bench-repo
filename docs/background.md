# 背景：AIファーストなプログラミング言語とは

2025年末〜2026年前半にかけて、「人間ではなくAIエージェントがコードを書く前提」で
設計された言語が相次いで登場した。コミュニティ編集のカタログ
[agentlanguages.dev](https://agentlanguages.dev) はこれらを追跡しており、
設計思想で大きく3つの流派に分けている。

- **構文派**：問題は表現の曖昧さにある。構文から曖昧さを排除する
- **検証派**：問題は意味的な正しさにある。コンパイラが機械的に検証できる契約を持たせる
- **オーケストレーション派**：言語ではなくエージェント協調の問題だと再定義する

## このベンチマークで扱う4言語

| 言語 | 流派 | 一言で |
| --- | --- | --- |
| Zero (zerolang) | 検証＋構文 | 人間が読めるソースから意味グラフを作り、AIはグラフを編集する |
| MoonBit | 検証派 | 最古参・最成熟。生成中に文法/型エラーを抑制する仕組みを持つ |
| NanoLang | 検証＋構文 | 最小設計。全関数にテスト必須。中核仕様を形式的に証明済み |
| Vera | 検証派 | 全関数に契約必須＋形式検証。変数名を廃止 |

既存言語（JavaScript / Python）は、モデルが大量の訓練データで熟知している
「対照群」として比較に加える。新言語は訓練データが乏しいハンデを、
設計の良さでどこまで補えるかが見どころ。

※スター数や成熟度は時期により変動する。最新はカタログや各リポジトリを参照のこと。
