# ai-lang-benchmark

「人間ではなく **AIエージェントがコードを書く前提** で設計された言語」と、既存の汎用言語を、
**同じ課題をAIに解かせて比較する** ためのベンチマークキットです。

同じ仕様を各言語に渡し、AIに「生成 → 実行 → 失敗したら修正 → 採点」まで自走させ、
比較レポートを自動生成します。

## 比較対象の言語

| 言語 | 位置づけ |
| --- | --- |
| JavaScript | 既存の汎用言語（訓練データが豊富な対照群） |
| Python | 既存の汎用言語（訓練データが豊富な対照群） |
| [Zero (zerolang)](https://zerolang.ai/) | グラフファースト。AIがグラフを編集する |
| [MoonBit](https://www.moonbitlang.com/) | 最古参・最成熟。AIフレンドリーな汎用言語 |
| [NanoLang](https://github.com/jordanhubbard/nanolang) | 最小設計。全関数にテスト必須 |
| [Vera](https://veralang.dev/) | 契約＋形式検証。変数名を廃止 |

> なぜこれらの言語なのか、という背景は [docs/background.md](docs/background.md) を参照。

## 課題一覧

| 問題 | 内容 | 測れること |
| --- | --- | --- |
| [問題1：JSONパーサ](problem1-json-parser/) | 境界条件込みのJSONパース | 異常系の処理漏れ、エラー修正力 |
| [問題2：銀行口座](problem2-bank-account/) | 残高は負にならない等の不変条件 | 契約・検証の効き目 |
| [問題3：バグ修正](problem3-bug-fix/) | バグ入りコードの自己修復 | 構造化エラーの有利さ |

3問とも独立に実行できます。問題3は問題2の実装を流用するため、
問題2 → 問題3 の順で進めるとスムーズです。

## 文法ガイドについて（公平な比較のため）

新言語4つは訓練データが少ないため、仕様だけ渡す（パターンA）と不利になります。
そこで各言語の**公式文法ガイドを与える条件**（パターンB）でも測り、
「訓練データの無さを設計で補えているか」を見ます。

`guides/` に各言語の**公式ガイドの取得方法**をまとめてあります。
ガイド本文を丸写しせず取得方法だけ置いているのは、常に最新・正確を保ち、
リポジトリを軽量にするためです。Zero と Vera は公式に機械可読ガイドを提供しています
（Zero: `zero skills get zero-language` / Vera: `https://veralang.dev/SKILL.md`）。

## 使い方

1. このリポジトリをクローンする
2. 比較したい言語の処理系をインストールする（任意。無い言語は自動でスキップされます）
3. 対象の問題フォルダをAIコーディングエージェント（Claude Code など）に渡し、こう指示する

> `AGENT_INSTRUCTIONS.md` を読んで、その通りにベンチマークを最後まで自動実行し、`REPORT.md` を出力してください。

エージェントが6言語を順に処理し、`results/` に成果物を、ルートに `REPORT.md` を出力します。

## 注意

- Zero / MoonBit / NanoLang / Vera は実験的な言語です。処理系が無い環境では
  その言語は `SKIPPED` として記録され、ベンチマークは残りの言語で続行されます。
- 結果は **使用モデル・プロンプト・試行回数で変わります。**
  レポートには必ず実行条件が明記されます。フェアな比較のため、
  新言語には「仕様のみ」と「文法ガイド付き」の2条件が用意されています。
