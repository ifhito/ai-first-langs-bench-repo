# bank-account：銀行口座

各言語でAIに銀行口座モジュールを実装させて比較します。
詳しい題材の意図はリポジトリ直下の [README](../README.md) と
このフォルダの `spec.md` を参照してください。

## このフォルダの中身

| ファイル | 役割 |
| --- | --- |
| `AGENT_INSTRUCTIONS.md` | エージェントへの実行指示（メイン） |
| `spec.md` | 課題の仕様（全言語共通） |
| `testcases.md` | テストケース |
| `prompts/` | 言語ごとのプロンプト雛形 |
| `results/` | 実行結果がここに溜まる |
| `REPORT.md` | 最終レポート（エージェントが出力） |

## 出題意図（人間向け・⚠️ 生成プロンプトには絶対に含めない）

> この節は採点者・読者のための背景説明です。**`spec.md` や `prompts/` には入れず、コード生成側には渡しません**。
> 「各言語が不変条件をどう表現すべきか」という想定解を生成側に与えると、
> 「モデルが自力でそこに到達したか」を測れなくなる（計測の汚染）ためです。

「残高は負にならない」という1つの不変条件を、各言語がどう表現するかを見る。

- JavaScript / Python：明示的なチェックを書き忘れると、言語は止めてくれない
- Vera：契約（事前条件・事後条件）として宣言でき、検証器が機械的に確認できる**かもしれない**
- NanoLang：必須のテストで不変条件を担保**しうる**
- Zero：構造化された型・エラーで不正操作を弾け**るかもしれない**

→ 同じルールが「人間（AI）が書く防御コード」になるか「言語が保証する契約」になるか、
その差が出るかどうか自体を観測対象とする（どちらになるかは事前に決めつけない）。

## 実行方法

このフォルダをAIコーディングエージェントに渡し、こう指示します。

> `AGENT_INSTRUCTIONS.md` を読んで、その通りにベンチマークを最後まで自動実行し、`REPORT.md` を出力してください。
