# 仕様：銀行口座（問題2）

口座残高を管理するモジュールを実装する。
この問題の核心は「**残高は決して負にならない**」という不変条件を、コード全体で守れるか。
この仕様は全言語で完全に同一とする。仕様にない機能は追加しないこと。

## 操作

- `deposit(amount)`：入金する
- `withdraw(amount)`：出金する
- `balance()`：現在の残高を返す

各言語の慣習に従ってよい（クラス／構造体／モジュール等、表現は自由）。

## 満たすべきルール（不変条件・事前条件）

- 残高の初期値は 0
- 入金・出金の金額は **正の整数のみ**（0以下は拒否してエラー）
- 残高を超える出金は拒否してエラー（残高は変えない）
- どの操作の前後でも、残高は **常に 0 以上**
- 金額は整数で扱う（小数・通貨単位は考えない）

## エラーの扱い

- ルール違反の操作（負の金額、残高超過の出金など）は、
  各言語の慣習に従ってエラーとして扱う（例外／エラー値）。
- エラーが起きても **残高は不正な状態にならない** こと。

## 合格条件

- 一連の操作を流したあと、残高が一度でも負になったら不合格
- ルール違反の操作が「成功」として通ってしまったら不合格
- 正常な操作は正しく残高に反映されること
