# プロンプト雛形：Vera

## パターンA（仕様のみ）
```
あなたは Vera のコードを書く開発者です。
以下の仕様を満たすJSONパーサを1つ実装してください。

# 制約
- 仕様に書かれた境界条件をすべて満たすこと
- 仕様にない機能は追加しないこと
- 完全に動作する単一の実装を出力すること
- 出力はコードのみとし、説明は不要

# 仕様
# 仕様：JSONパーサ（問題1）

文字列を受け取り、JSONとして解釈して値を返す関数を実装する。
この仕様は全言語で完全に同一とする。仕様にない機能は追加しないこと。

## 関数の形

- 入力：JSON文字列を1つ
- 出力：パース結果の値、または「エラー」を表す結果
  - 各言語の慣習に従ってよい（例外を投げる／エラー値を返す等）

## 対応する型

- null
- 真偽値（true / false）
- 数値（整数・小数。指数表記 `1e5` などは対象外でよい）
- 文字列（ダブルクォート。エスケープは `\"` `\\` `\n` の3つに対応すれば十分）
- 配列
- オブジェクト（キーは文字列）

## 正常にパースすべき入力

- `null` → null
- `true` / `false` → 真偽値
- `42` → 整数 42
- `3.14` → 小数 3.14
- `"hello"` → 文字列 hello
- `"a\"b"` → 文字列 a"b（エスケープ）
- `[]` → 空配列
- `{}` → 空オブジェクト
- `[1, 2, 3]` → 配列
- `{"a": 1, "b": true}` → オブジェクト
- `{"a": [1, {"b": null}]}` → ネスト構造
- 値の前後の空白 ` 42 ` → 42（トリムして解釈）

## エラーにすべき入力（クラッシュせずエラーを返すこと）

- 空文字列 `` → エラー
- 末尾カンマ `[1, 2, ]` → エラー
- 閉じ括弧不足 `[1, 2` → エラー
- 閉じ括弧不足 `{"a": 1` → エラー
- 不正な数値 `01` → エラー
- 不正な数値 `1.` → エラー
- 不正な数値 `.5` → エラー
- 未知のトークン `foo` → エラー
- クォート閉じ忘れ `"abc` → エラー

## 合格条件

- 正常系：正しい値を返す
- 異常系：プログラムが異常終了せず、エラーとして扱う
```
